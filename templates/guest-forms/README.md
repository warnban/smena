# Шаблоны бланков для гостей

Положите сюда свои `.docx` файлы с **точными именами**:

| Файл | Бланк |
|------|--------|
| `personal-data-consent.docx` | Согласие на обработку персональных данных *(встроенный шаблон: `npm run generate:pd-consent`)* |
| `hotel-rules.docx` | Правила проживания в гостинице *(встроенный шаблон: `npm run generate:hotel-rules`)* |
| `refund-form.docx` | Бланк возврата денежных средств *(встроенный шаблон, можно пересоздать: `node scripts/generate-refund-form-template.mjs`)* |
| `hotel-services-contract.docx` | Договор на оказание гостиничных услуг *(встроенный шаблон: `npm run generate:hotel-contract`)* |
| `hotel-contract-amendment.docx` | Доп. соглашение к договору (изменение срока) *(встроенный шаблон: `npm run generate:hotel-contract-amendment`)* |

Путь в проекте:

```
C:\smena-crm\templates\guest-forms\
```

## Как писать переменные в Word

Да, используйте **фигурные скобки** — CRM подставит значения автоматически:

```
{guest_fio}
{hotel_name}
{stay_period}
```

### Важно при редактировании в Word

1. Пишите каждый плейсхолдер **одним фрагментом**, без пауз посередине.  
   Неправильно: `{guest` + `_fio}` (Word разбивает текст — подстановка не сработает).
2. Надёжный способ: наберите `{guest_fio}` в Блокноте и вставьте в Word одной строкой.
3. Не используйте «умные» кавычки «» внутри скобок — только `{` и `}`.


## Настройки отеля

В **Настройки → Отель** заполните:

- **Юридическое название** — подставляется в `{hotel_legal_name}`
- **Сайт** — в `{hotel_website}`

После замены файлов перезапуск dev-сервера не обязателен — шаблоны читаются при каждой печати.

## Сложные бланки (таблицы, бланк возврата)

1. **Переменные ставьте в отдельные ячейки таблицы**, а не в середину длинного текста.
2. Оставьте в ячейке запас места — длинное ФИО не должно ломать сетку.
3. Если Word «ломает» `{guest_fio}` на части — наберите переменную в Блокноте и вставьте одной строкой.
4. Для проверки: **Предпросмотр** в CRM → должно выглядеть как в Word. Если в Word после «Скачать Word» всё ок, а в предпросмотре нет — сообщите (редкий случай).
5. Эталонная проверка — открыть скачанный `.docx` в Microsoft Word.

## Бланк возврата (`refund-form.docx`)

Шаблон создаётся скриптом `scripts/generate-refund-form-template.mjs` и уже лежит в папке.

**Юридические отличия от «старых» бланков:**

- Нет удержания произвольной «комиссии 15%» при оплате картой — возврат тем же способом (ст. 22 ЗоЗПП).
- Срок возврата — **10 календарных дней**, а не «2 рабочих дня».
- Пересчёт скидки при досрочном выезде: без привязки к конкретным срокам в бланке — по **фактическому** сроку и действующим тарифам (скидочная категория → более низкая категория → полный тариф).
- Нет штрафа «за последнюю ночь без уведомления за 24 часа» — только оплата фактически оказанных услуг.

Разметка — таблицы Word, переменные в отдельных ячейках, чтобы при печати и предпросмотре ничего не «разъезжалось».

## Согласие на ПДн (`personal-data-consent.docx`)

Шаблон создаётся скриптом `scripts/generate-personal-data-consent-template.mjs`.

Содержит блоки, требуемые ст. 9 152-ФЗ: оператор, перечень данных, цели, действия, сроки, отзыв, подпись. Передача третьим лицам — **только в МВД** для миграционного учёта; хранение — для базы гостей и в сроки, установленные законом.

Перед печатью заполните у гостя в CRM: паспорт, кем/когда выдан, адрес регистрации, дата рождения.

## Правила проживания (`hotel-rules.docx`)

Шаблон: `scripts/generate-hotel-rules-template.mjs` → `npm run generate:hotel-rules`.

**Юридические правки относительно старого текста:**

- Миграционный учёт: передача в МВД — **в течение 24 часов с заселения**, а не «с момента пересечения границы».
- Досрочный выезд: пересчёт по **фактическому тарифу/скидке**, без удержания «за одни сутки».
- Выселение за нарушения: возврат **неиспользованной предоплаты** по закону и правилам возврата, а не «без возврата денег».
- Отказ в размещении — только на **законных основаниях**, не «без объяснения причин».
- Залог за ключ — **размер на ресепшн**, не фиксированные 300 ₽ в тексте (можно указать в прейскуранте).

## Договор на гостиничные услуги (`hotel-services-contract.docx`)

Шаблон: `scripts/generate-hotel-services-contract-template.mjs` → `npm run generate:hotel-contract`.

Соответствует ГК РФ (глава 53), ЗоЗПП (ст. 8–10, 17, 22, 32), ПП РФ № 1853 (правила оказания гостиничных услуг). Подставляются стоимость и оплата из бронирования: `{booking_amount}`, `{booking_paid}`, `{booking_balance}`, номер договора `{contract_number}`.

## Дополнительное соглашение (`hotel-contract-amendment.docx`)

Шаблон: `scripts/generate-hotel-contract-amendment-template.mjs` → `npm run generate:hotel-contract-amendment`.

Формируется **автоматически** после изменения срока проживания (продление или сокращение) в карточке брони. Содержит ссылку на основной договор, прежний и новый период, пересчёт стоимости, порядок доплаты/возврата (ст. 450, 452 ГК РФ; ст. 8–10, 22, 32 ЗоЗПП).

Дополнительные переменные: `{amendment_number}`, `{contract_date}`, `{prev_stay_period}`, `{new_stay_period}`, `{prev_booking_amount}`, `{new_booking_amount}`, `{amount_delta}`, `{night_delta}`, `{change_type}`, `{change_summary}`.
